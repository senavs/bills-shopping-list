# Feature Suggestions

## High Impact (daily use)

### 1. Bill splitting among people
Assign items to people (diners/shoppers), split shared items equally, and see per-person totals including tax. This is the #1 pain point for restaurant groups.

### 2. Separate tip vs. tax
Currently there's a single `taxPercentage`. Restaurants often have a mandatory service charge + optional tip. Two separate fields would make this more realistic.

### 3. Item discounts/coupons
Add a percentage or fixed discount per item (e.g., "buy 2 get 10% off"). Common in supermarkets.

### 4. Search and filter
Search across lists and filter items within a list (by name, selected/unselected, section). As lists grow, this becomes essential.

### 5. Spending history with dates
Add timestamps to lists. Show a monthly spending summary chart — "you spent R$X on groceries in June."

## Medium Impact (quality of life)

### 6. More currencies
EUR, GBP, or a custom input field instead of the BRL/USD hardcode.

### 7. Item notes
A small description field per item (e.g., "brand preference", "check expiry date").

### 8. Templates / recurring lists
Save a list as a template and reuse it weekly (common grocery runs).

### 9. Sort items
Auto-sort by name, price, or checked/unchecked status (one-tap sort button).

### 10. Undo on delete
Instead of permanent deletion for items, show a brief toast with "Undo" (the list/section already has confirm dialogs, but items don't).

## Lower Priority (nice-to-have)

### 11. Cloud sync
Optional Google Drive or simple cloud backup beyond JSON export.

### 12. Unit types
kg, lb, liter, pack — so `quantity` has context.

### 13. Store/restaurant name per list
Associate a venue and optionally save favorites.

### 14. Different tax rates per item
Real groceries have items taxed at 0%, 5%, 12%, etc. One global rate is a simplification.
